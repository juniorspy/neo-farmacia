# Session 2026-04-09-01

**Stage**: Planning / Architecture
**Goal**: Define the complete project architecture and documentation structure.

## Context Gathered
- Studied neo colmado (conecta2) — Android app + Firebase + n8n ordering system
- Studied whatsapp-service — middleware between Evolution API and Firebase
- Studied n8n workflow template (main template_001.json) — 5 AI agents for colmado ordering
- Studied backend-neocolmado — Spring Boot admin backend with Thymeleaf dashboard

## Decisions Made
- **No Firebase**: Project is fully independent from neo colmado
- **n8n for AI**: Keep n8n as AI orchestrator (too complex for pure code). Microservice is the bridge.
- **Stack**: Fastify + TypeScript (API), Next.js (Dashboard), Odoo 17 (Inventory), MongoDB (Chats), Redis (State)
- **Web-only**: No native app. Dashboard replaces both Android app and admin backend functionality.
- **Roadmap order**: Odoo → Microservice → n8n → WhatsApp → Dashboard → POS Sync → Production

## Key Architecture Findings
- Need per-conversation mutex (Redis SETNX) to prevent concurrent n8n executions
- Need idempotency check on message IDs to prevent duplicate orders
- Need handover check at both ingress AND egress
- Odoo JSON-RPC can be slow — cache in Redis (5min TTL)
- Minimum 16GB VPS RAM for full stack
- n8n has concurrency limits — acceptable for 1-5 tenants
